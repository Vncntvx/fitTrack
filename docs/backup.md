## 系统概述

### 功能特性

备份系统支持以下核心功能：

- **多存储后端**：支持 WebDAV 和 S3 兼容存储
- **数据完整性**：使用 SHA-256 校验和确保备份完整性
- **多配置管理**：支持配置多个备份目标
- **默认配置**：可设置默认备份目标，简化操作流程
- **备份历史**：自动记录每次备份的元数据

### 支持的平台

| 平台 | 备份支持 | 恢复支持 |
|------|----------|----------|
| Android | 支持 | 支持 |
| Web | 支持 | 支持 |

### 备份内容

备份文件包含以下数据：

- 运动记录（workouts）
- 力量训练条目（strength exercises）
- 用户设置（settings）
- 备份配置（backup configurations）

## 备份架构

### 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                      BackupService                          │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │   WebDAV     │  │     S3       │  │  BackupVerifier  │  │
│  │  Provider    │  │  Provider    │  │                  │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                            │
                    ┌───────┴───────┐
                    ▼               ▼
            ┌─────────────┐   ┌─────────────┐
            │   WebDAV    │   │  S3 Storage │
            │   Server    │   │             │
            └─────────────┘   └─────────────┘
```

### 数据流

```
备份流程：
数据库 → 导出(JSON, 含版本) → AES-256-GCM 加密 → SHA-256 校验(明文+密文) → 上传 → 记录历史

恢复流程：
下载 → 元数据校验 → 密文 SHA-256 验证 → 解密(AES-GCM) → 明文 SHA-256 验证 → JSON 解析 → 数据库导入
```

### 文件命名规范

备份文件使用以下命名格式：

```
workout_backup_YYYYMMDD_HHMMSS.json
```

示例：
- `workout_backup_20240322_143052.json`

## WebDAV 配置

### 适用场景

WebDAV 适合以下使用场景：

- 个人用户部署私有云存储
- 局域网内备份（如 NAS 设备）
- 需要简单配置的场景

### 配置参数

| 参数 | 必填 | 说明 |
|------|------|------|
| 显示名称 | 是 | 配置的标识名称 |
| 服务端点 | 是 | WebDAV 服务器 URL |
| 路径 | 否 | 备份存储的子目录路径 |
| 用户名 | 是 | WebDAV 认证用户名 |
| 密码 | 是 | WebDAV 认证密码 |
| 设为默认 | 否 | 是否设为默认备份目标 |

### 配置步骤

1. **打开备份设置页面**
   - Android：设置 → 备份与恢复 → 添加配置
   - Web：管理员控制台 → 备份管理 → 添加配置

2. **选择 WebDAV 类型**

3. **填写配置信息**

   ```
   显示名称：我的 NAS 备份
   服务端点：https://nas.example.com/webdav
   路径：/workout-backups
   用户名：your-username
   密码：your-password
   ```

4. **测试连接**
   - 点击"测试连接"按钮验证配置
   - 确保服务器可访问且认证信息正确

5. **保存配置**

### 常见 WebDAV 服务端

#### Nextcloud

```
端点格式：https://your-domain.com/remote.php/dav/files/username
路径：/workout-backup
```

#### ownCloud

```
端点格式：https://your-domain.com/remote.php/webdav
路径：/workout-backup
```

#### 群晖 NAS (Synology)

```
端点格式：https://your-nas.com:5006/webdav
路径：/workout-backup
```

#### 威联通 NAS (QNAP)

```
端点格式：https://your-nas.com/webdav
路径：/workout-backup
```

## S3 配置

### 适用场景

S3 适合以下使用场景：

- 需要高可靠性的企业用户
- 使用云存储服务的用户
- 需要跨区域备份的场景

- **Amazon S3**：AWS 官方 S3 服务
- **MinIO**：自托管 S3 兼容存储
- **Backblaze B2**：兼容 S3 API 的云存储
- **其他兼容服务**：任何 S3 API 兼容的存储服务

| 设为默认 | 否 | 是否设为默认备份目标 |

## 恢复备份

### 恢复前注意事项

⚠️ **警告**：恢复操作会覆盖当前的所有运动记录数据。建议在恢复前：

1. 先执行一次当前数据的备份
2. 确认要恢复的备份文件完整无误
3. 了解恢复操作不可逆

### Android 平台恢复

1. 打开应用，进入"设置"页面
2. 点击"备份与恢复"
3. 选择"从备份恢复"
4. 浏览备份历史列表
5. 选择要恢复的备份记录
6. 点击"恢复"按钮
7. 确认恢复操作
8. 等待恢复完成
9. 应用将自动重启以加载新数据

### Web 平台恢复

#### 通过管理界面恢复

1. 访问管理员控制台
2. 导航至"备份管理"页面
3. 查看备份历史列表
4. 找到要恢复的备份记录
5. 点击"恢复"按钮
6. 确认恢复操作

#### 通过 API 恢复

```bash
# 恢复指定备份
curl -X POST http://localhost:8080/api/backup/restore \
  -H "Content-Type: application/json" \
  -d '{
    "providerType": "webdav",
    "targetPath": "/workout_backup_20240322_143052.json",
    "checksum": "abc123..."
  }'
```

### 恢复流程说明

恢复操作执行以下步骤：

1. **下载备份文件**：从远程存储获取备份数据
2. **解析并校验元数据**：校验 `payloadFormat`/加密版本/算法信息
3. **验证密文校验和**：使用 SHA-256 校验加密载荷完整性
4. **解密备份文件**：使用 AES-256-GCM 解密（认证标签失败则立即终止）
5. **验证明文校验和**：再次使用 SHA-256 校验导入前明文数据
6. **解析 JSON**：将备份文件解析为数据对象
7. **清空现有数据**：删除当前的运动记录和力量训练数据
8. **导入备份数据**：将备份数据写入数据库
9. **保留用户设置**：现有设置不会被覆盖

## 备份验证

### 加密与校验机制

备份系统默认使用 **AES-256-GCM** 对备份 JSON 明文加密，并结合 **SHA-256** 做双层完整性验证：

```dart
// 1) 计算明文校验和（用于恢复前最终校验）
final plaintextChecksum = sha256.convert(plaintextBytes).toString();

// 2) 加密后计算密文校验和（用于下载后先验校验）
final encryptedChecksum = sha256.convert(encryptedBytes).toString();
```

### 验证流程

1. **备份时计算**
   - 导出数据库后，先计算明文 SHA-256
   - AES-256-GCM 加密后再计算密文 SHA-256
   - 将算法版本、密文校验和、明文校验和写入备份记录元数据

2. **恢复时验证**
   - 若元数据声明为加密载荷，必须按加密流程恢复，不允许静默回退为明文
   - 先校验密文 SHA-256，再执行解密
   - 解密后校验明文 SHA-256；任一步失败均拒绝恢复

### 兼容性说明

- `payloadFormat = encrypted-v1`：严格按加密恢复流程执行，不允许明文回退。
- 历史未加密备份：仅在记录未声明加密格式时，按旧版明文 SHA-256 流程恢复。
- 导入 JSON 时要求显式版本（`version` 或 legacy `exportVersion`），无版本数据会被拒绝导入。

### 手动验证备份文件

```bash
# 计算本地文件的 SHA-256 校验和
sha256sum workout_backup_20240322_143052.json

# macOS
shasum -a 256 workout_backup_20240322_143052.json
```

### 元数据文件

备份时会生成对应的元数据文件（`.meta.json`）：

```json
{
  "filename": "workout_backup_20240322_143052.json",
  "size": 15234,
  "checksum": "a1b2c3d4e5f6...",
  "checksumAlgorithm": "SHA-256",
  "createdAt": "2024-03-22T14:30:52.123Z",
  "note": null
}
```

## 故障排除

### 连接问题

#### 无法连接到 WebDAV 服务器

**症状**：测试连接失败，提示网络错误

**排查步骤**：

1. 检查端点 URL 是否正确
   ```
   错误：https://example.com/webdav/
   正确：https://example.com/webdav
   ```

2. 验证网络连通性
   ```bash
   curl -I https://your-webdav-server.com
   ```

3. 检查认证信息
   - 确认用户名和密码正确
   - 检查账户是否有 WebDAV 访问权限

4. 检查服务器配置
   - 确认 WebDAV 服务已启用
   - 检查防火墙设置

#### S3 连接失败

**症状**：测试连接失败，提示 403 或 404 错误

**排查步骤**：

1. 检查端点格式
   ```
   AWS：https://s3.amazonaws.com
   MinIO：https://minio.example.com:9000
   ```

2. 验证密钥权限
   - 确认 Access Key 和 Secret Key 正确
   - 检查密钥是否有存储桶读写权限

3. 检查存储桶名称
   - 确认存储桶存在
   - 检查存储桶名称拼写

4. 检查 CORS 配置（Web 平台）
   ```xml
   <CORSConfiguration>
     <CORSRule>
       <AllowedOrigin>*</AllowedOrigin>
       <AllowedMethod>GET</AllowedMethod>
       <AllowedMethod>PUT</AllowedMethod>
       <AllowedMethod>POST</AllowedMethod>
       <AllowedHeader>*</AllowedHeader>
     </CORSRule>
   </CORSConfiguration>
   ```

### 备份失败

#### 备份文件上传失败

**症状**：备份过程中断，提示上传错误

**解决方案**：

1. 检查存储空间
   - 确认远程存储有足够空间

2. 检查文件权限
   - 确认有写入目标目录的权限

3. 检查网络连接
   - 确保网络稳定
   - 尝试切换网络环境

4. 减小备份频率
   - 如果数据量过大，考虑分批备份

#### 校验和错误

**症状**：恢复时提示"校验和不匹配"

**解决方案**：

1. 重新下载备份文件
   - 可能是下载过程中数据损坏

2. 检查存储介质
   - 远程存储可能存在数据损坏

3. 使用其他备份
   - 选择其他时间点的备份文件

### 恢复失败

#### 恢复后数据不完整

**症状**：恢复成功但缺少部分运动记录

**排查步骤**：

1. 检查备份文件完整性
   ```bash
   # 检查 JSON 格式
   cat workout_backup_*.json | python -m json.tool
   ```

2. 查看备份历史
   - 确认选择的备份包含所需数据

3. 尝试其他备份
   - 选择不同时间点的备份进行恢复

#### 应用崩溃或无法启动

**症状**：恢复后应用无法正常运行

**解决方案**：

1. 清除应用数据（Android）
   - 设置 → 应用 → 运动追踪 → 存储 → 清除数据

2. 重新安装应用
   - 卸载后重新安装
   - 重新配置并恢复备份

3. 联系支持
   - 如果问题持续，提供备份文件供分析

### 性能问题

#### 备份速度过慢

**优化建议**：

1. 使用局域网 WebDAV
   - 比公网 S3 速度更快

2. 选择就近区域
   - S3 选择物理距离近的区域

3. 优化网络环境
   - 使用稳定的 Wi-Fi 连接
   - 避免高峰时段备份

#### 大数据量备份

如果数据库超过 100MB：

1. 定期清理历史数据
2. 考虑增量备份方案
3. 使用压缩存储（后续版本支持）

### 常见问题 FAQ

**Q: 可以同时配置多个备份目标吗？**
A: 可以。应用支持配置多个 WebDAV 和 S3 备份目标，但同一时间只能有一个默认配置。

**Q: 备份文件会加密吗？**
A: 当前版本备份文件以 JSON 格式存储，未加密。建议通过 HTTPS 传输并存储在受保护的存储服务上。

**Q: 可以跨平台恢复吗？**
A: 可以。Android 和 Web 平台使用相同的备份格式，可以互相恢复。

**Q: 备份会包含应用设置吗？**
A: 备份包含用户设置数据，但恢复时不会覆盖当前设置，以保护现有配置。

**Q: 如何删除旧备份？**
A: 目前需要手动登录到 WebDAV 或 S3 存储管理界面删除。应用内删除功能将在后续版本中添加。

## 安全建议

1. **使用 HTTPS**：始终使用 HTTPS 协议传输备份数据
2. **保护密钥**：不要共享 S3 密钥，定期轮换密钥
3. **定期验证**：定期测试备份恢复流程，确保备份可用
4. **多地备份**：建议配置多个备份目标，提高数据安全性
5. **访问控制**：限制存储服务的访问权限，使用最小权限原则
